组内git最简使用参考

Posted on 2019-03-28

git 和 github简介（及小组最简使用策略）

关于建立源仓库（组员建立自己仓库的操作）

  • 建立repo(比如varpour,建议勾选init readme)
  • 打开git bash
  • 进入项目目录
  • 比如 git clone https://github.com/varpour/your_repo 到本地
  • 进入your_repo
  • 随意新创建一个文件夹或者文件之类
  • git config –global user.name ‘你的用户名’
  • git config –global user.email ‘你的注册邮箱’
  • git add .
  • git commit -m ‘your comment about this commit’
  • git push origin master
  • 刷新页面github页面可以看到你的文件

关于clone他人源仓库(比如varpour)到本地

  • user(你的用户)登录
  • 在varpour的项目页面(比如keras_learning这个项目)右上部找到“Fork”按钮，将该仓库 Fork 到你自己的账户中
  • 用git clone 命令git clone https://github.com/user/keras_learning 到本地
  • 做各种本地修改，添加，然后
  • git config user.name ‘你的用户名’
  • git config user.email ‘你的email’
  • git add .
  • git commit -m ‘your comment about this commit’
  • git push origin master
  • push之后，就会看到user的页面下面有新添加的东西，但是源仓库里不变

关于提交PR(Pull Request)

  • 在user页面顶部选择 Pull request 标签：然后点击 New pull request 按钮
  • 确认无误之后，填写一些comment信息，点击 Create pull request 按钮
  • 然后这个pr就到达varpour/keras_learning页面(就是源仓库的页面，即项目负责人可以看到)
  • 随后，varpour，即这个仓库的所有者，会被通知有人提交了 Pull request
  • 在varpour确认这个 Pull request 修改是正确的、可接受的之后，就会按 Merge pull request 按钮，confirm merge按钮，一个代码的修正就完成
  • 注意：这时候varpour的源仓库是完整的状态

关于同步源仓库

  • cd keras_learning进入目录
  • git remote add upstream https://github.com/varpour/keras_learning.git（添加远端源repo）
  • 同步远端源repo
    • git fetch upstream（没变化本地）
    • git rebase upstream/master（本地有变化，同步成功，github没变化）
  • git push origin master（推送github）
  • 后续可以进行修改，继续提交PR

注意

  • 提交 Pull request 的时候，最佳策略如下：

    • 尽量每次只提交一个小修改(或者比如添加一个文件)
    • 提交时尽量简短且清楚地说明修改原因
    • 耐心等待回复

  • Fork过来的仓库在本地可以随意修改，但不建议当作用来提交Pull request的版本。需要向别人提交Pull request的时候，必须重新获取一个当前最新版本到本地，在其基础上修改，然后PR。
